Withdrawal

The objective of this experiment is to evaluate the use of LLMs to shape trajectories by suggesting 3D poses based on natural language input. This hypothesis is answered by experimenting with the withdrawal and change of objective methods. For the first experiment, the withdrawal method is tested with a variety of prompts. These prompts express positions relative to the user's location or other objects on the board. The second experiment, akin to the first, aims to gauge satisfaction with the change of objective based on user input. Each experiment is evaluated through a user base study of x participants who rate their satisfaction with GPT-4's responses for the two functions. The task involves the robot picking up a cube positioned on the left side of the user and relocating it between the user's hands. Throughout the experiment, the user's left-hand serves as the closest point and thus represents the obstacle from which the robot attempts to withdraw. Trajectory points (without any withdrawal) are recorded along with the speed of the robot and safety indexes. Based on this data, three distinct trajectories are computed. Two types of prompts are subsequently tested: commands relative to the user and commands relative to objects on the table. For instance, a command relative to the user might be: “Move to the right”, while a command relative to objects could be: “Move closer to the stack”. For each prompt, the three withdrawal methods are tested for three coordinates along the trajectory (9 trajectories per prompt). One at the beginning of the trajectory, one in the middle, and one at the end. On the video, one can see an example of a withdrawal trajectoriy for the point in the middle of the trajectory for the prompt “Move to the left”. For the examples below, withdrawal 1 (in orange) is the original withdrawal method presented by Garcia et al., withdrawal 2 (in red) is the method where GPT suggests the position of withdrawal, and withdrawal 3 (in green) is a method where GPT suggests a new parking position (mix between GPT-4 and withdrawal 1).

Change Objective

The robot is stopped while picking up a cube, which is not displayed to the user as depicted in the figure above. There are 25 cubes on the table from which the robot can choose to change objective based on the user input. Similar to the first experiment, the user's input can consist of commands either relative to the user or relative to objects in the scene. For instance, a command relative to the user might be: “Don't take cubes on the right”, while a command relative to other objects could be: “Take cubes that are close to the robot”.
